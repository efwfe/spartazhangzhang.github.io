网页分析的几项重要指标

访问次数

花一点时间浏览了网页内容 然后离开，这个过程称为会话。

独立访客

  网站分析尽可能使其接近真实的访客的人数；
技术上，在用户访问网站的时候此人的浏览器创建一个独立的cookie,包括了独立的匿名字符串，不包含用户的私人信息；
根据这一点每次用户访问网站的时候系统通过识别cookie来判断独立用户的数量；
从这个角度来看，cookie的生存时长的大小能够影响到网站数据的分析情况；

网站停留时间

  网站停留时间计算能够分为以下几种种情况：

  1. 用户根据页面跳转访问多个页面，能够通过两次请求的时间差来计算请求的大致时间，以为最后一次退出的时间并不能计算到；

  2. 用户访问的是单个页面，只是浏览了一次就离开网站的话，这样就只有第一次请求的时间戳，离开的时长依然无法得知，但是可以通过其他手段提供缺失值；

  3. 用户多页面访问的时候，在时间点上，用户的请求时间是交错的，并不是完全按照网站结构层次来请求，针对这种情况，大部分网站分析会独立计算页面停留时长最后来计算整个浏览时长；

跳出率(除了博客)

  主要针对拿些来到网站没有任何操作行为的访客，从技术角度来定义： 网站上单个一面访问所占的会话比例。

  衡量网站的流量最大的反向链接的跳出率，不仅能够看出引流的渠道，能够衡量渠道的质量。

  衡量关键词的跳出率，可能是关键词的选择存在问题，或者是对应网站的着落也存在问题。

退出率

  如同字面意思来看，主要是表名访客从网站离开的情况。

  虽然每个用户来到网站最后都会离开，但是从哪个页面退出，也并不能够指出哪些页面需要改进，或者缺少用户需要的信息，从这个角度来看的话，退出率并不是一个很好的指标，那么他又有啥用呢？

  我的理解是：他能够衡量一部分没有任何操作的用户，没做任何操作就从页面退出的比率；

转化率

   当然是分析中最重要的指标

   一个问题是，使用独立访客做分母，还是使用所有的访客做分母，对于这个的选择取决于对于网站的理解，怎么样的分析模式更适合公司的业务，不同的转化率计算对后续的反向链接，市场活动，关键词效率都有很大的影响。

参与度

  网站分析中，很多对参与度指标的定义不清，导致很多并不使我们需要的指标，一是，这个指标可能是参与度的一个方面，二是隐藏其真正要衡量的目标。

  试想一个情况，如果我在网站上找我需要的资料花了十分钟，而某人在该网站上看了十分钟需要的资料，从停留时间来看参与度是一样的，这一方面来看，网站分析之所以不能很好的分析参与度是因为它只能衡量参与度的程度，而不能区分参与类型。

  针对这个情况，可以从访客次数、访客频率、网站停留时间、网站注册率、订阅、提交评论等指标侧面了解参与度的程度；

  每个网站都有自己的独特目标，了解竞争者的使用指标也是很有好处。此外，必须对自身独特业务选择合适的衡量指标；还可以考虑针对一些定性问题，进行访问调研，电话调研。

logging 日志模块的基本概念

日志一共分成5个等级，从低到高分别是：

  1. DEBUG
  2. INFO
  3. WARNING
  4. ERROR
  5. CRITICAL

说明:

  • DEBUG：详细的信息,通常只出现在诊断问题上
  • INFO：确认一切按预期运行
  • WARNING：一个迹象表明,一些意想不到的事情发生了,或表明一些问题在不久的将来(例如。磁盘空间低”)。这个软件还能按预期工作。
  • ERROR：更严重的问题,软件没能执行一些功能
  • CRITICAL：一个严重的错误,这表明程序本身可能无法继续运行
    这5个等级，也分别对应5种打日志的方法： debug 、info 、warning 、error 、critical。默认的是WARNING，当在WARNING或之上时才被跟踪。
