  • list

    Python列表在传统的计算机科学意义上并不是真正意义上的列表，这就解释了为什么追加比插入高效得多。一个经典的列表-所谓的链接列表-被实现为一系列节点，每个节点(除了最后一个节点)保持对下一个节点的引用。一个简单的实现可能如下所示:

通过指定所有节点来构造列表。

python的列表类型的底层实现有一点不同。一个列表不是几个相互引用的独立节点，一个列表基本上是一个单一的、连续的内存块，通常被称为数组。这导致了与链接的一些重要区别。例如，虽然迭代列表的内容对两种类型都同样有效(除了链接列表中的一些开销)，但是在数组中直接访问给定索引上的元素要有效得多。这是因为可以计算元素的位置，并且可以直接访问正确的内存位置。

渐近表示法自19世纪末开始使用(有一些变化)，是分析算法和数据结构的重要工具。其核心思想是将我们分析的资源(通常是时间，有时也是内存)表示为一个函数，其参数是输入大小。例如，我们可以有一个运行时间t(N)=2.4n7的程序。
立即出现了一个重要的问题：这里的单位是什么？我们测量运行时间(以秒或毫秒为单位，还是用位或兆字节表示问题的大小)似乎很琐碎。不过，有些令人惊讶的答案是，这不仅是琐碎的，而且实际上也不会影响我们的结果。我们可以测量时间(以木星年为单位)和问题大小(以公斤为单位)。

渐近表示法由一群以希腊字母写成的运算符组成。最重要的和我们将要使用的唯一运算符是O(原为Omicron，但现在通常称为“biooh”)、(Omega)和(Theta)。o运算符的定义可用作其他两个字母的基础。表达式o(G)，用于某些函数g(N)，表示一组函数，如果函数f(N)满足以下条件：存在一个自然数N0和一个正常数c，则f(N)在这个集合中f (n) <= cg(n),换句话说，如果允许我们调整常数c(例如，通过在不同速度的机器上运行算法)，函数g最终会比F大(即在N0)
