Web Analytic II

发表于 2018-03-30 | 分类于 web data

实践操作

基础分析思路

   无论是何种报表，都需要思考为什么要选用这些指标，并且在此过程中进行深刻的辨析。

细分数据

  大部分网站需要的不是分析，只是报表而已。因为大多数的分析只是一个粗糙的总结性的结论。而细分数据是分析的基础，不仅能够提升对网站业务的理解，还能在更深层次上看出数据中隐藏的问题，对公司的决策层的沟通和决策都起到很好的作用。

关注用户行为而非总体行为

  针对数据进行有效划分数据，针对不同的用户群体进行分析，了解是什么其中的原因，可以进行有效内容分析和行为分析。

站内搜索分析

1.站内搜索引擎的使用
2.衡量搜索质量
3.细分搜索效率

真实性验证
跟踪cookie值

  1.临时cookie，能够有效的将各种点击事件进行分组归类；2.永久cookie用来识别来访的唯一浏览器。

数据抽样

  最好的策略是在手机所有数据在运行查询和报表时进行抽样，次选择是在网页手机数据的时候对网页进行部分数据抽样。

历史数据

  点击层面的数据按季度保存一年即可，会话层面的数据最好尽可能久的保存。，历史数值在其他领域很宝贵，在网站分析则逐渐下降。

数据矫正清单

  1.服务器日志和Js加码的统计不具可比性
  2.cookie
  3.跟踪代码的安装问题
  4.session的时效性
  5.url的结构，网站链接的请求存在很多参数的情况
  6.广告链接的设置链接的标记
  7.数据抽样的精度调整
  8.加码的顺序，一般是加到最后防止页面加载没有完成用户就跳转的记录。
