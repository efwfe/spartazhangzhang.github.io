SQLAlchemy Lazy属性

select (True ==‘select’)

  当首次访问属性时，使用单独的SELECT语句或简单的多对一引用的关系映射获取来加载元素。

immediate

  使用单独的SELECT语句，在加载父项时加载项目时加载子元素，或对简单的多对一引用进行关系映射。

joined (False == ‘joined’)

   应该使用“JOIN”或“LEFTJOIN”在与父连接相同的查询中积极地加载项目。查询根据内联接参数决定的。

subquery

   子查询-当加载父级时，应该加载一个项目，使用一个附加的SQL语句，为每个请求的集合发出一个连接到原始语句的子查询。

selectin

   在加载父级时，应使用“一个或多个附加SQL语句”加载项目，这些语句会向即时父对象发出联接，使用IN子句指定主键标识符。

noload (None == ‘noload’)

   不应在任何时间发生负载。这是为了支持“只写”属性，或者以特定于应用程序的方式填充的属性。

raise

   不允许偷懒加载；访问该属性，如果其值尚未通过急于加载加载，将引发无效请求错误。这种策略可以在对象被加载后从它们所连接的会话中分离出来时使用。

raise_on_sql

   不允许发出SQL的惰性加载；访问属性，如果其值尚未通过快速加载加载，则如果懒惰负载需要发出SQL，则将引发无效请求错误。如果懒惰负载可以从身份映射中提取相关值，或者确定它不为任何值，则加载该值。当对象将与附加会话保持关联时，可以使用该策略，但是应该阻止额外的SELECT语句。

dynamic

   属性将返回所有读取操作的预配置查询对象，在迭代结果之前，可以应用其他过滤操作。有关详细信息，请参阅动态关系加载程序部分。
